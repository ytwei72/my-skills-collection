# 出版排版标准参考手册

本文档为 `edu-typeset` skill 的详细参考，供处理复杂排版任务时查阅。

---

## 出版印刷通用标准

### 字体规范（中文出版物）

| 层级 | 字体 | 字号 | 字形 |
|-----|------|------|-----|
| 书名/封面大标题 | 黑体 | 22–26pt | 粗体 |
| 篇章标题（一级）| 黑体 / 方正大标宋 | 16pt | 粗体 |
| 节标题（二级）| 黑体 / 微软雅黑 | 14pt | 粗体 |
| 小节标题（三级）| 楷体 / 微软雅黑 | 12pt | 粗体 |
| 正文 | 宋体 | 10.5–11pt | 常规 |
| 注释/脚注 | 宋体 | 9pt | 常规 |
| 页眉/页脚 | 宋体 | 9–10pt | 常规 |

> 英文内容对应字体：标题用 Times New Roman，正文用 Times New Roman 或 Georgia。

### 行距与间距（中文出版物）

| 场景 | 行距 |
|-----|------|
| 正文 | 1.5 倍（约 22–24pt）|
| 注释 | 1.2 倍 |
| 标题上间距 | 正文行距 × 1.5 |
| 标题下间距 | 正文行距 × 0.8 |

在 docx.js 中，行距用 `line` 属性，单位为 twip 的 1/100（240 = 单倍行距，360 = 1.5 倍，276 ≈ 1.15 倍）。

### 页边距（A4）

| 位置 | 标准值 |
|-----|-------|
| 上 | 2.0–2.5 cm（1134–1418 DXA）|
| 下 | 2.0–2.5 cm |
| 左 | 2.5–3.0 cm（1440–1728 DXA）|
| 右 | 2.0–2.5 cm |

> DXA（Document eXchange Absolute）：1 英寸 = 1440 DXA，1 cm ≈ 567 DXA。

## 教辅材料专项规范

### 内容层级识别

读取原始文档后，识别以下层级：

```
1. 书名/系列名          → 封面块（coverBlock）
2. 前言/目的说明         → 目的表格（purposeTable）
3. 章节标题（Chapter N）→ 章节块（chapterBlock）
4. 节标题（Section）    → 小节标题（sectionHead）
5. 提示语/说明框         → 提示框（hintBox）
6. 英语原文（含填空）    → 英语正文段落（engPara）
7. 中文译文             → 中文译文段落（cnPara）
8. 答案                 → 答案块（answerBlock），仅答案
9. 解析                 → 答案块（answerBlock），答案+解析
10. 语法规则总结         → 规则框（ruleBox）
11. 记忆口诀/顺口溜      → 口诀框（rhymeBox）
12. 对比/辨析表          → 两列对照表（twoColumnTable）
```

### 填空题处理规范

原文中的填空格式多样（如 `__`、`______`、`[ ]`），排版时统一保留为：
- 在正文中：`__________` 加【提示语】标注（如 `【1. call 变形 谓语/非】`）
- 在练习说明中注明题号与提示

**禁止**在清洗阶段删除连续下划线（勿使用 `.replace(/_{2,}/g, '')` 一类规则），否则会去掉填空线。

#### Pandoc / Markdown 转义（常见故障）

经 `pandoc docx -t markdown` 提取时，下划线会被写成 `\_`（避免被当成 Markdown 强调）。若未还原就写入 `TextRun`，成品 docx 会显示为 `\_\_\_\_\_\_`（可见反斜杠）。

**必做**：写入 docx 前对全部提取文本调用模板中的 `normalizeExtractedText()`（`\_` → `_`，并处理 `\*`、`` \` `` 等常见转义）。

```javascript
const { normalizeExtractedText, engPara } = require('.../template-en-edu.js');
const line = normalizeExtractedText(pandocLine);  // \_\_\_\_ → ____
engPara(line);
```

`engPara` / `cnPara` 内部已调用该函数；自定义 `clean()` 或手写 `TextRun` 时须自行调用。

**优先**：教辅含大量填空时，尽量用 **docx 直接抽文本**（python-docx、OOXML）；若必须用 pandoc，中间稿仅作结构参考，填空字符必须经 `normalizeExtractedText`。

### 答案与解析分离规则

- 若原文答案与解析在同一段落，拆分为 `answerBlock` 的两行（答案行 + 解析行）
- 若只有答案无解析，`answerBlock(num, answer)` 省略第三参数
- 多个答案块之间用 `spacer(60, 60)` 分隔，视觉上不过于紧凑

## 附录生成规则

### 附录应包含哪些内容

附录是对**原文知识内容的提炼**，不是原文复制。判断标准：

- ✅ 对学习者有复习查阅价值的规则/表格
- ✅ 分散在正文各处、集中呈现更有用的内容
- ✅ 易混淆的概念对比
- ❌ 不添加原文没有出现的知识
- ❌ 不重复正文已有的段落

### 英语教辅附录标准组件

1. **不规则动词表**（如原文涉及不规则动词）
   - 格式：四列表（原形 / 过去式 / 过去分词 / 中文）
   - 使用 `twoColumnTable` 扩展为四列

2. **词形变化规则汇总**（如原文涉及 -ed/-ing/-ly/-s 等变形）
   - 格式：规则框（`ruleBox`）+ 示例

3. **语法规则速查表**（如原文涉及多个语法点）
   - 格式：两列对照表（规则名 / 说明+示例）

4. **易混淆词汇对比表**（如原文有词汇辨析内容）
   - 格式：三列表（词A / 词B / 区别说明）

## docx npm 包常见陷阱

| 问题 | 正确做法 |
|-----|---------|
| 背景色变黑 | `ShadingType.CLEAR`，不用 `ShadingType.SOLID` |
| 表格错位 | 同时设 `columnWidths`（表级）和每格 `width`（格级），均用 DXA |
| 换行失效 | 不用 `\n`，用独立 `new Paragraph()` |
| 列表符号乱码 | 不用 `•` 字符，用 `LevelFormat.BULLET` + numbering config |
| 横向规则用表格 | 不行，用段落 `border.bottom` 实现分割线 |
| 页码不显示 | Footer 中用 `PageNumber.CURRENT`，不能写死数字 |
| 中文字体不生效 | 同时设 `font`（英文）和 `font`（中文用 eastAsia）时注意 docx.js 的字体属性 |
